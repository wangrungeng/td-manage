# TDengine 可视化管理工具执行计划

## 技术路线

- Monorepo：`pnpm workspace`
- 前端：`Vue 3 + TypeScript + Vite + Pinia + Vue Router + Element Plus`
- 后端：`Node.js + TypeScript + Fastify`
- 本地库：`SQLite + better-sqlite3`
- TDengine：`@tdengine/websocket`

## 关键实现

1. 创建项目骨架和基础配置。
2. 后端实现环境配置、SQLite 初始化、默认角色、JWT 鉴权、RBAC 权限。
3. 后端实现连接配置加密保存和连接测试。
4. 后端实现 TDengine 元数据查询、数据分页查询。
5. 后端实现新增、编辑、删除的 SQL 预览和执行接口。
6. 后端实现 SQL 控制台风险识别和审计记录。
7. 前端实现登录、初始化、主布局、连接管理、数据浏览、SQL 控制台、用户角色、审计日志。
8. 补充 README 和环境变量示例。

## 数据安全策略

- 用户密码使用 bcrypt hash。
- TDengine 密码使用 `APP_SECRET` 派生 AES-256-GCM 加密保存。
- 所有 TDengine 写操作由后端生成最终 SQL。
- 删除执行前先预览，执行时要求 `DELETE`。
- SQL 控制台写语句执行时要求 `CONFIRM`。
- 默认禁止 `DROP`、`ALTER`、`CREATE`、`IMPORT`、`UPDATE`。

## 性能策略

- 表数据查询必须提供时间范围。
- 默认分页 100 条。
- 最大分页 1000 条。
- SQL 控制台 SELECT 未带 LIMIT 时自动追加 `LIMIT 100`。

## 风险

- TDengine 客户端包的实际 API 可能与当前封装存在差异，需要安装依赖后根据官方包类型校正。
- TDengine 单行编辑依赖同时间戳覆盖写入语义，不支持时应禁用编辑。
- 删除不可恢复，只能通过预览、确认和审计降低误操作风险。

## 建议验证

1. 安装依赖：`pnpm install`。
2. 类型检查：`pnpm typecheck`。
3. 启动服务：`pnpm dev`。
4. 浏览器访问前端，初始化管理员。
5. 使用测试 TDengine 环境创建连接并验证查询、新增、编辑、删除流程。
