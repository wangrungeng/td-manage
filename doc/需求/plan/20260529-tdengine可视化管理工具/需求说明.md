# TDengine 可视化管理工具需求说明

## 原始需求

在新云平台开发过程中，手动编写 SQL 删除、修改 TDengine 数据不方便，缺少适合个人使用的开源管理软件。希望在当前空项目中开发一个可视化工具，专门用于编辑、删除、修改、新增 TDengine 库表内容，并带有工具自身登录功能。

## 目标

- 提供本地 Web 可视化页面。
- 提供工具自身登录、多用户、多角色权限控制。
- 使用 SQLite 保存工具自身系统数据。
- 支持自建 TDengine 3.x 连接管理。
- 支持库表浏览、分页查询、新增、覆盖式编辑、精确删除。
- 所有写操作执行前必须 SQL 预览和二次确认。
- 所有 TDengine 写操作记录完整审计。

## 范围

### 包含

- 登录和初始化管理员。
- 用户角色管理。
- TDengine 连接增删改查和连接测试。
- 数据库列表、表结构和数据查询。
- 新增数据 SQL 预览和执行。
- 同时间戳覆盖写入式编辑。
- 按 `database + table + ts` 精确删除。
- SQL 控制台基础执行和风险拦截。
- 审计日志查询。

### 不包含

- TDengine Cloud 适配。
- 桌面安装包。
- 生产多人部署。
- TDengine 用户权限、节点、集群管理。
- 数据迁移、批量修改、批量删除。
- 删除后重插式编辑降级。

## 约束

- 默认只监听 `127.0.0.1`。
- 不保存明文 TDengine 密码。
- 不在前端或日志暴露明文密码。
- 查询优先使用时间范围和分页限制，避免无边界扫描。
- 未经用户明确授权，不连接真实服务器或数据库。

## 验收口径

- 可启动前后端并通过浏览器访问。
- 首次使用可初始化管理员并登录。
- 可创建 TDengine 连接配置并进行测试。
- 可查询库表数据。
- 新增、编辑、删除前能看到 SQL 预览。
- 删除必须输入 `DELETE`。
- 写 SQL 必须输入 `CONFIRM`。
- viewer 角色不可写数据。
- 写操作审计日志可查询。
